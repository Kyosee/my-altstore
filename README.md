# Kazumi — Unofficial AltStore / SideStore Source

An **unofficial** AltStore / SideStore subscription source that auto-tracks the
iOS builds published by the upstream [Predidit/Kazumi](https://github.com/Predidit/Kazumi)
project. It exists because upstream ships an unsigned `.ipa` in every release but
does not host an AltStore source ([Predidit/Kazumi#1516](https://github.com/Predidit/Kazumi/issues/1516)).

This repository is **not affiliated with, endorsed by, or maintained by** the
Kazumi authors. It only rebuilds a manifest that points at their official files.

## Add the source

Use this URL in AltStore / SideStore → **Sources → Add Source**:

```
https://raw.githubusercontent.com/Kyosee/kazumi-altstore/master/apps.json
```

## How it works

A daily [GitHub Actions workflow](.github/workflows/update.yml) regenerates
[`apps.json`](apps.json) from upstream's GitHub releases. For each release it:

1. Finds the `Kazumi_ios_*_no_sign.ipa` asset.
2. Downloads that `.ipa` and computes its SHA-256 **locally**, then cross-checks
   it against the digest GitHub computed server-side. A mismatch aborts the run.
3. Reads `CFBundleIdentifier`, `CFBundleVersion`, and `MinimumOSVersion` from the
   `.ipa`'s embedded `Info.plist` — nothing app-specific is hardcoded.
4. Keeps the 5 most recent versions so you can roll back.

## Supply-chain safety

This source is built to be **auditable** and to give tampering nowhere to hide:

- **No re-hosted binaries.** Every `downloadURL` points *directly* at
  `github.com/Predidit/Kazumi/releases/...`. This repo never stores or proxies an
  `.ipa`, so there is no copy here to swap for a poisoned build.
- **Verifiable hashes.** Each version's `sha256` is computed from the real
  upstream file on every run; AltStore verifies it at install time. If upstream's
  file and the manifest hash ever disagree, installation fails.
- **Pinned actions.** The workflow pins `actions/checkout` to a full commit SHA,
  not a moving tag, and pulls in no other third-party actions. The update script
  uses only the Python standard library — no `pip install`, no external packages.
- **Minimal permissions.** The workflow token is scoped to `contents: write`
  (only to push the regenerated `apps.json`) and read-only against upstream.
- **Everything is diffable.** The manifest is regenerated by committed code you
  can read in full; each automated update lands as its own commit.

If you don't trust this repo, you don't have to: fork it, read
[`scripts/update_source.py`](scripts/update_source.py) (~290 lines, single file),
and run it yourself.

## Manual regeneration

```bash
python3 scripts/update_source.py
# optional: export GITHUB_TOKEN=... to raise the API rate limit
```
